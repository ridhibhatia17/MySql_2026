## 1. Basic Filtering with WHERE

The WHERE clause is used to extract only those record that fulfill a specified condition.

- Equality Check:

```sql
SELECT * FROM books WHERE category = 'Technology'; -- All books with technology will be printed.
```

- Numeric Comparison: Filter by values using operators like <, >, <=, >=, <>.
    Example: Find books with a price less than $30.

```sql
SELECT title, price FROM books WHERE price < 30.00;
```

- Date Comparison: SQL allows direct comparison on data types.
    Example: Books published on or after March 1st, 2023

```sql
SELECT title, publication_date FROM books
WHERE publication_date >= '2023-03-01';
```

## 2. Logical Operators

Combine multiple conditions to refine your search:

- AND: Returns records if all conditions are true.

Example:

```sql
SELECT * FROM books WHERE category = 'Technology' AND price < 30;
```

- OR: Returns records if any condition is true.

Example:

```sql
SELECT * FROM books WHERE category = 'Technology' OR price < 30;
```

- Combining Operators: Use parentheses () to define the order of evaluation (e.g., specific categories AND a price limit).

Example:

```sql
SELECT * FROM books WHERE (category = 'Technology' OR category = 'Mystery') AND price < 25;
```

- NOT: Displays records where the condition is not true.

Example:

```sql
SELECT * FROM books WHERE NOT category = 'Technology';
```

## 3. Handling NULL Values

Standard comparison operators (like =) do not work with NULL because it represents an "unknown" value.

- Find Nulls: Use IS NUL.

Example:

```sql
SELECT * FROM books WHERE author IS NULL;
```

- Find Non-Nulls: Use IS NOT NULL.

Example:

```sql
SELECT * FROM books WHERE author IS NOT NULL;
```

Note: Using WHERE author = NULL will return no results because the comparison evaluates to "Unknown" rather than True.

## 4. Pattern Matching with LIKE

Search for specific patterns in text data using Wildcards:

- % (Percent): Represents zero, one or multiple characters.
    - %SQL%: Contains "SQL" anywhere.

Example:

```sql
SELECT * FROM books WHERE title LIKE '%SQL%';
```

    - 'The%': Starts with "The".

Example:

```sql
SELECT * FROM books WHERE title LIKE 'The%';
```

- Case Sensitivity: By default, LIKE is case-insensitive. Use LIKE BINARY to enforce case sensitivity.

Example:

```sql
SELECT * FROM books WHERE title LIKE BINARY '%SQL%';
```

- _ (Underscore): Represents exactly one single character.

Example:

```sql
SELECT * FROM books WHERE author LIKE '_ohn%';
```

## 5. Advanced Range & List Filtering

- BETWEEN: Filters values within a specific range (inclusive).

Example:

```sql
SELECT * FROM books
WHERE price BETWEEN 20 AND 30;
```

- IN: Allows you to specify multiple values in a WHERE clause.

Example:

```sql
SELECT * FROM books
WHERE category IN ('Technology', 'Mystery');
```

## 6. Subqueries in WHERE

You can use the result of one query inside another:

- Comparison with Subquery: Find items with a price greater than the average price.

```sql
SELECT * FROM books WHERE price > (SELECT avg(price) FROM books);
```

- Subquery with IN: Filter categories based on a dynamic list generated by another query.

```sql
SELECT * FROM books WHERE category in (
        SELECT category FROM books WHERE in_stock > 20
);
```

```sql
-- Find all books published in 2023 that cost less than the average book price
SELECT * FROM books
WHERE YEAR(publication_date) = 2023 AND price < (SELECT AVG(price) FROM books);

-- List all technology books with "data" in the title that have more than 50 copies in stock
SELECT * FROM books
WHERE category = 'Technology' AND title LIKE '%data%' AND in_stock > 50;

-- Find books that are either in the technology category with the price > $30 or in the Mystery category with price < $20
SELECT * FROM books
WHERE (category = 'Technology' AND price > 30) OR (category = 'Mystery' AND price < 20);

-- List all books where the author's name contains either 'son' or 'th' and were published after March 2023
SELECT * FROM books
WHERE (author LIKE '%son%' OR author LIKE '%th%') AND publication_date > '2023-03-31';
```
